# cap2jianying 版本更新说明

> 说明：本项目为 Windows 桌面录屏工具（可选麦克风录音）。FFmpeg 需要用户自行安装或在界面中选择 `ffmpeg.exe`。

---

## v0.2.0（2026-04-24）

### ✨ 新增

- **清晰度选择**：新增「高 / 中 / 低」三档清晰度（圆点单选），并持久化保存到本地配置。

### 🧩 优化

- **界面布局**：
  - 「麦克风」与「清晰度」同一行展示，界面更紧凑。
  - 「刷新麦克风」按钮替代原“麦克风”标题，更一致，也让 FFmpeg 路径输入区域更宽。
  - 「清晰度」标题改为按钮风格外观，整体一致性更好。
- **录制清晰度**：按清晰度档位自动映射 H.264（`libx264`）编码参数（`preset/crf`），在清晰度与性能之间更可控。
- **FFmpeg 下载入口**：下载按钮改为打开 FFmpeg 官方下载页（更稳定，包含 Windows 预编译来源入口）。

### 📦 构建/发布

- **打包脚本增强**：支持 release/test 命名、版本记忆，并支持非交互参数，便于自动化打包。
- **发布包自动整理**：打包完成后自动移除产物中的 `logs/`，并在上级目录生成同名 `.zip`（便于直接上传到 Release）。

## v0.1.0（2026-04-22）

## 亮点
- 支持桌面录屏（Windows）+ 可选麦克风录音
- 录制完成后可一键「保存到剪映草稿」
- 支持在界面内选择 `ffmpeg.exe`，解决双击运行时 PATH 不一致问题

## 本次更新内容
- 修复迁移后打包脚本路径：自动定位 `scripts/` 与 `assets/`，不再依赖上级目录结构
- 源码运行更稳定：导入剪映前自动将 `scripts/vendor` 加入 `sys.path`，并补齐 `requirements.txt`
- 编码兼容：移除脚本输出中的 emoji；外部调用强制 UTF-8，避免 Windows 控制台 GBK 编码异常
- 打包修复：移除 `collect-all tkinter`，解决 Tcl/Tk 版本冲突导致的 exe 启动失败
- 文档补充：新增 conda / venv 两种依赖安装方式，并说明启动方式与环境的对应关系

## 使用说明（打包版）
1. 解压后运行 `VideoRecorderOnly.exe`
2. 机器需已安装 FFmpeg（或在界面中选择 `ffmpeg.exe`）
3. 录制完成后点击「保存到剪映草稿」

## 已知限制
- FFmpeg **不内置**在安装包内，目标机器需要可用的 `ffmpeg.exe`
- PyInstaller 产物可能被部分杀软误报：可尝试添加信任/放行
